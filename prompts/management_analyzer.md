# 老板 Management Style 分析 Prompt

## 任务

你将收到 {name} 的原材料（邮件、聊天记录、文档等）。
从中提取他的管理风格与行为模式。

**原则：只提取与管理相关的内容，忽略闲聊。不要推断，有依据才写。**

---

## 提取维度

### 1. 管理风格画像

**维度A：控制欲**
- 高控型：事事过问、审批链长、关键决策必须经过他
- 结果导向型：只问结果，不管过程，但会追问细节
- 放任型：给方向但不跟进（不一定是好事，可能最后秋后算账）

**维度B：沟通频率**
- 轰炸型：消息秒回、随时拉会、24小时在线、深夜也轰炸
- 定时型：只在周会/1v1时沟通，但消息必须及时回复
- 选择性忽略型：看心情回复，但关键时刻一定会出现

**维度C：反馈风格**
- 表扬极少，批评极多（通常比例 1:9）
- 批评时对事不对人 vs 对人不对事
- 先骂后安抚 vs 冷处理 vs 阴阳怪气
- 表扬的方式：真诚表扬 vs 敷衍式表扬 vs 附带条件的表扬

**维度D：决策模式**
- 快速拍板型：当场追问，当场拍板
- 犹豫不决型：反复讨论，要你提供更多信息
- 甩锅型：让你做决定，错了是你的，对了是他决策英明

**维度E：责任归属**
- 有担当型：出了问题先承担，再追责
- 甩锅型：出了问题永远不是他的责任
- 混合型：看情况，好的情况是他负责，不好的情况是下属负责

---

### 2. 开会行为

从会议相关记录中提取：
- 开会频率（每天/每周几次）
- 会议时长（5分钟 vs 2小时）
- 会议准时度（从不迟到/偶尔迟到/经常迟到/迟到也不等）
- 参会要求（必须发言/旁听即可/缺席要请假）
- 会议形式（站着开/坐着开/视频/线下）
- 是否需要提前准备材料

```
输出格式：
开会频率：{N}次/周
平均时长：{N}分钟/次
准时要求：{从不迟到/迟到会怎样}
会议准备：{需要材料/不需要/简单说说就行}
会议形式：{站着/坐着/视频}
典型会议：
- 早会（9:00）：{描述}
- 周会（周三14:00）：{描述}
- 临时会：{描述}
```

---

### 3. 汇报要求

- 周报/月报格式要求（结构、长度、必须包含的内容）
- 汇报频率（日报/周报/随时）
- 汇报结构要求（背景-行动-结果 / 直接说问题 / 结论先行）
- 对数据的重视程度（必须量化 / 尽量量化 / 定性也行）
- 禁止出现的词汇（"还行"/"差不多"/"尽力了"等）

```
输出格式：
汇报要求：
- 频率：{N}次/周
- 格式：{描述}
- 结构：{描述}
- 必须包含：{列表}
- 禁止出现：{列表}
- 数据要求：{必须量化/尽量量化/定性也行}
```

---

### 4. OKR 与考核

- OKR 激进程度（挑战型：目标100%=没挑战 / 保守型：目标80%=正常完成）
- 考核标准（量化考核 / 模糊评价 / 领导主观判断）
- 绩效评定维度（产出/过程/态度/潜力等的权重）
- 晋升标准（公开 / 半公开 / 暗箱操作）
- 淘汰机制（末位淘汰 / 稳定型 / 观察期）

```
输出格式：
OKR风格：{挑战型/保守型/适中}
考核标准：{量化/模糊/主观}
晋升机制：{公开/暗箱/混合}
淘汰机制：{有/无/观察期}
绩效维度权重：
- 产出：{N}%
- 过程：{N}%
- 态度：{N}%
- 其他：{N}%
```

---

### 5. 向上管理可能性

评估：
- 是否可以提前预判他的要求
- 是否可以通过调整汇报方式减少被喷
- 他的哪些行为是可以"应对"的
- 他的哪些行为是"无解"的

```
输出格式：
可预判行为：
- {行为1}：应对方式：{方法}

可应对行为：
- {行为1}：应对技巧：{技巧}

无解行为：
- {行为1}（只能熬过去）

最佳策略：
1. {策略1}
2. {策略2}
3. {策略3}
```

## 标签翻译规则

将用户填写的标签翻译为具体管理行为：

| 标签 | 翻译为管理行为规则 |
|------|------------------|
| **开会狂魔** | 每天3个会起步，每个会必须有议程和准备，否则会被追问"没准备来干嘛" |
| **深夜轰炸** | 晚上11点-凌晨1点消息是常态，已读不回第二天会被追问"工作量不饱和？" |
| **周末不休** | 周六周日也会@你，不回复会被记住，回复了会被觉得"应该的" |
| **CR 杀手** | 任何输出物必须经他CR，通常打回2-3轮，每轮都追问细节 |
| **数据控** | 没有数据的判断 = 废话，会被当众质疑"你的依据是什么" |
| **细节控** | 一个标点错都会发飙，PPT字体不对会当场退回 |
| **PPT 侠** | 一切皆需PPT，10页以内讲不清楚问题 = 能力不行 |
| **焦虑型** | 频繁追问进度，任何延期都会放大成危机，永远觉得deadline不够早 |
| **冷血型** | 犯错后不会安抚，直接追责，从不照顾情绪 |
| **情绪化** | 心情好什么都行，心情不好鸡毛蒜皮也能发火，不可预测 |
| **容错率为零** | 任何小错误都会在全员会上被点名，从不私下提醒 |
| **微操狂魔** | 明明授权了还是会插手，说"我只是看看"然后开始改代码 |
| **甩锅型** | 出了问题永远不是他的责任，永远能找到理由 |
| **当众羞辱** | 会在群里/会上公开批评，不留情面，让人下不来台 |
| **阴阳怪气** | 不直接骂，但讽刺拉满，听完血压升高 |
| **信息差玩家** | 故意不把信息同步清楚，看你怎么应对 |
| **OKR激进派** | OKR永远是挑战值，完成70%算优秀，100%说明目标定低了 |
| **目标压缩机** | 永远把你的时间线砍一半，说"应该够了吧" |
| **邮件控** | 邮件格式要求极高，没有结论的邮件会被退回重写 |
| **已读乱回型** | 消息已读不回是常态，只有他主动找才算响应 |
| **秒回强迫症** | 消息5分钟内没回就开始追问，显示在线却不回复会被认为态度有问题 |

## 输出要求

- 语言：中文
- 没有信息的维度：标注 `（原材料不足，建议追加邮件/聊天记录）`
- 有原文依据的结论：加引号标注原话
- 输出结果直接用于生成 management.md，要求具体可执行
- 必须包含"向上管理建议"
