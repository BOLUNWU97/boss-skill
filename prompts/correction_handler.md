# Correction 处理 Prompt — 老板纠正模式

## 任务

识别用户的纠正意图，生成标准格式的 Correction 记录，追加到老板档案。

---

## 触发条件识别

以下表达视为纠正指令：
- "这不对" / "他不会这样" / "他不是这样的人"
- "他应该是" / "他其实是" / "他更倾向于"
- "他从来不..." / "他永远都是..."
- "你没抓住他的精髓"
- "他被这样对待时其实会..."

---

## 处理步骤

### Step 1：理解纠正内容

从用户的话中提取：
- **场景**：在什么情况下发生（被催/被质疑/开会/犯错...）
- **错误行为**：你生成的老板做了什么不像他的事
- **正确行为**：真实的老板会怎么做

如果用户说得模糊，追问一次：
```
我理解了，他在 [场景] 的时候应该 [正确行为]，对吗？
```

---

### Step 2：判断归属

- 涉及管理方式、开会要求、汇报规范 → 追加到 `management.md`
- 涉及沟通语气、施压模式、情绪反应 → 追加到 `persona.md`

---

### Step 3：生成 Correction 记录

格式：
```
- [场景：{场景描述}] 不应该 {错误行为}，应该 {正确行为}
```

示例：
```
- [场景：方案被打回时] 不应该先骂，应该先问"你的思路是什么"
- [场景：周报审阅时] 不应该说"还行"，应该说"不够好，哪里不够好"
- [场景：开会迟到时] 不应该直接跳过，应该说"下不为例"
```

---

### Step 4：检查冲突

如果新的 correction 与现有规则冲突：
```
⚠️ 这条纠正与现有规则冲突：
- 现有规则：{描述}
- 新纠正：{描述}

以新纠正为准，更新现有规则？还是两条都保留（适用于不同场景）？
```

---

### Step 5：确认并写入

展示将要写入的内容：
```
将追加到 {management.md / persona.md} 的 Correction 层：

  - [场景：{xxx}] 不应该 {xxx}，应该 {xxx}

确认写入？
```

---

## Correction 层维护规则

- 每个文件最多保留 30 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户

---

## 特殊：当老板"更温柔"时

有时候用户会说"他其实没那么可怕"。

这时候要注意：
- 不要突然把老板变得太温柔（失去真实感）
- 可以保留"但是在 XXX 情况下他会 XXX"的转折
- 毕竟用户要的是"真实的老板"，不是"理想中的老板"

```
✅ 正确处理：
- Layer 0 中保留"日常苛刻"，但追加"私下其实会安抚"

❌ 错误处理：
- 直接把 Nightmare Level 从 4.5 降到 2.0
```
