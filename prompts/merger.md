# 增量 Merge Prompt — 老板进化模式

## 任务

你将收到：
1. 现有的 `management.md` 内容
2. 现有的 `persona.md` 内容
3. 新的原材料（邮件、聊天记录、截图等）

你的任务是分析新内容，更新老板的苛刻档案。

**原则：只追加增量，不覆盖已有结论。发现矛盾时让用户决定。**

---

## Step 1：分类判断

| 新内容类型 | 归入 |
|-----------|------|
| 管理风格变化、新的开会行为 | → management.md |
| 新的施压话术、苛刻行为 | → persona.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- **补充**：新内容增加了细节 → 直接追加
- **确认**：新内容确认了现有信息 → 忽略（不重复）
- **矛盾**：新内容与现有矛盾 → 输出冲突提示

```
⚠️ 发现冲突：
- 现有：{描述}
- 新发现：{描述}
- 来源：{文件名/时间}

建议：[保留现有 / 更新为新内容 / 两者都保留]
请决定。
```

---

## Step 3：生成更新 Patch

对 `management.md`：
```
=== management.md 更新 ===

[追加到"开会行为"节]
- 新开会规矩：{描述}

[追加到"汇报要求"节]
- 新汇报要求：{描述}

[无更新] 或 [以上章节有更新]
```

对 `persona.md`：
```
=== persona.md 更新 ===

[追加到"Layer 2/施压口头禅"节]
- 新施压话术："{xxx}"

[追加到"Layer 4/管理施压"节]
- 新施压场景：{描述}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：确认并写入

展示更新摘要：
```
本次更新摘要：
- management.md：追加了 {N} 条新管理规则
- persona.md：追加了 {N} 条新施压行为
- 发现 {N} 处冲突，需要你确认

版本将从 v{N} 升级到 v{N+1}。
确认应用更新？
```
