## 🎯 一句话介绍

把让人窒息的老板蒸馏成 AI Skill，模拟他的苛刻审阅、追问、施压，帮你预判他会问什么、让你提前准备应对。

---

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🏢 **管理风格提取** | 从邮件/聊天记录提取管理风格、开会行为、汇报要求 |
| 😤 **苛刻人格还原** | 五层 Persona 还原老板的真实施压模式 |
| 📊 **Nightmare Level** | 量化评估老板的窒息程度（1-5星） |
| 🎯 **向上管理练习** | 对抗模式，练习如何应对老板的追问 |
| 📝 **方案/周报预审** | 模拟老板视角挑刺，提前准备好应对 |
| 🎤 **晋升面试模拟** | 模拟老板提问，练习晋升/跳槽面试 |
| 🔄 **持续进化** | 追加文件、对话纠正、版本回滚 |

## 🚀 快速开始

### 1. 创建你的老板

```
/create-boss
```

回答 3 个问题：
1. 老板称呼
2. 基本信息（公司/职级/管理风格）
3. 苛刻画像（MBTI/星座/让你崩溃的瞬间）

### 2. 导入原材料（可选）

```
原材料怎么提供？

  [A] 邮件导出（推荐）
      .eml / .mbox 格式
  
  [B] 聊天记录截图
      微信/钉钉/飞书截图
  
  [C] 直接粘贴/口述
      把老板的"名言"告诉我
```

### 3. 开始使用

```
/boss-{slug}           — 跟老板对线
/boss-{slug}-review   — 让老板审你的方案
/boss-{slug}-weekly   — 让老板审你的周报
/boss-{slug}-combat   — 向上管理练习
/boss-{slug}-interview — 晋升面试模拟
```

## 🎮 使用场景详解

### 场景1：写周报前
```
用户：帮我看看这周周报
老王：你的周报我看完了。
      第一个问题：'完成了XX'——数据在哪里？没有数据的完成度 = 没说。
      第二个问题：下周计划——时间线呢？没有具体时间的计划 = 废话。
      总结：不够好，重写。
```

### 场景2：汇报方案前
```
用户：帮我看看这个方案
老王：你的方案我看了。
      第一个问题：impact 是什么？（停顿）
      第二个问题：数据呢？有没有验证过？
      第三个问题：风险呢？失败了怎么办？
      第四个问题：谁来做？什么时候完成？
      总结：回去重写。下次来之前先问自己这7个问题。
```

### 场景3：被老板追问
```
用户：老板，这个需求太赶了，能不能多给两天？
老王：两天？（沉默3秒）
      你说'太赶'——依据是什么？试过哪些方案？
      如果你说'我尝试了X和Y还不够'——那我们可以谈。
      如果你只是说'太赶'——那我觉得你是在找借口。
      给你一天。今天下午5点前给我更新。
```

### 场景4：向上管理练习
```
老王：攻击开始。这个需求太慢了，为什么还没完成？
用户：我想解释一下，我们遇到了XX问题...
老王：你的回应打了6分。说'太赶了'——这是在找借口。
      更好的回应：'遇到了XX问题，我尝试了YY方案，建议ZZ方案，需要AA资源。'
      记住：永远不要只说太赶，要带着问题和解决方案来。
```

## 🔧 进化功能

### 追加模式
有新邮件/聊天记录时：
```
/update-boss {slug}
→ 追加分析 → 版本存档 → 增量更新
```

### 纠正模式
觉得老板"演得不像"时：
```
"他不会这样说"
→ 追加到 Correction 层 → 重新生成 SKILL.md
```

### 版本回滚
```
/boss-rollback {slug} v3
→ 恢复到历史版本
```
