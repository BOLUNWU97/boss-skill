# 老王 — Management Style

---

## 管理风格画像

### 控制类型
**高控型** — 事事过问，审批链长，重要决策必须经过他，技术方案要他点头，代码要他 review

### 沟通频率
**轰炸型** — 24小时在线，消息秒回，深夜轰炸是常态，期望下属也一样
- 工作日：随时可能 @你
- 深夜（22:00-01:00）：正常工作时间
- 周末：会发消息，回复是应该的，不回复会被记住

### 反馈风格
**批评极多，表扬极少** — 从不说"不错"，只说"还不够好"
- 90%的情况下会说"不够好"
- 9%的情况下会说"还行"
- 1%的情况下会说"进步了"
- 0%的情况下会说"很好"或"太棒了"

批评方式：当众点名，邮件抄送，群里公开，不私下提醒

### 决策模式
**快速拍板型** — 当场追问，当场拍板，不拖泥带水
但拍板后如果出问题，追责也很快速："当时不是说好了吗？"

### 责任归属
**甩锅型** — 出了问题永远不是他的责任：
- 需求没说清楚 = 产品的问题
- 技术实现有问题 = 开发的锅
- 测试没发现 = QA 的问题
- 流程有漏洞 = 运维的责任
- 他自己？他是决策者，决策没问题，执行有问题

## 开会行为

### 开会频率
**每天早会 + 每周周会 + 随时临时会**
- 早会：每个工作日 9:00-9:15，15分钟
- 周会：每周三 14:00-15:00，1小时
- 临时会：有突发情况随时开，不提前通知

### 会议时长
- 早会：严格15分钟，超时直接掐断
- 周会：1小时，按议程走，超时继续
- 临时会：有话则长，无话则短，但通常不超过30分钟

### 准时要求
**永远准时开始，从不等待迟到者**
- 迟到1分钟：口头警告
- 迟到3分钟：当众点名，迟到者的汇报被挪到最后或缩短
- 迟到5分钟以上：剥夺本次发言机会，下次早会补

### 开会必问问题（7连击）
1. "这个方案的 impact 是什么？"
2. "数据支撑在哪里？"
3. "有什么风险？"
4. "为什么选这个方案而不是另一个？"
5. "谁来做？什么时候完成？"
6. "如果失败了怎么办？"
7. "你保证能完成吗？你确定？"

### 会议雷区
- 没有议程的会不开（他会问"这个会的目的是什么"）
- 发言超过规定时间会被打断
- 讲不清重点会被当众质疑"你到底想说什么"
- 在群里公开反驳他会让他"记住你"
- 说"这不归我管"会被认为没有担当

### 典型会议

**早会（9:00-9:15）**
- 每人3句话汇报：完成了什么、遇到了什么、下一步
- 不允许讲过程，只讲结果
- 超时被直接打断："说重点"
- 站着开，提高效率

**周会（周三14:00）**
- 每人3分钟讲结果，不讲过程
- 用数据说话，没有数据等于没讲
- 会被追问7连击
- 会议结束后有 action item 和 deadline

**方案评审会**
- 永远在追问，永远在挑刺
- 很少当场说过，通常说"我再想想"
- "我再想想" = 基本没过，需要大改
- 通常打回2-3轮

## 汇报要求

### 汇报频率
**周报必须周五 18:00 前发**

### 汇报格式（严格遵守）

**必须包含：**
1. **结果**：完成了什么（用数字说话）
2. **问题**：遇到了什么（以及解决方案）
3. **下一步**：下周计划（带具体时间线）
4. **风险**：有什么风险（以及应对方案）

**禁止出现：**
- "还行"（必须说具体数字）
- "差不多"（量化，不接受模糊词）
- "尽力了"（只接受结果）
- "我觉得"（必须改成"数据显示"）
- "没有问题"（永远有风险，要主动说）

### 数据要求
**必须量化，没有数据的判断 = 废话**
- "提升了" → "提升了 X%，从 Y 到 Z"
- "做得不错" → "完成了 87%，超过目标 17%"
- "有进展" → "第3阶段完成，4/7测试用例通过"

## OKR 与考核

### OKR 风格
**挑战型** — OKR 永远是挑战值
- 目标 100% 完成 = 你没挑战
- 实际能完成 70% 就算优秀
- 100% 说明目标定低了
- 超过 100% 说明你之前藏了实力

### 考核标准
**量化 + 态度双考核**
- 产出结果：60%
- 数据支撑：20%
- 流程规范：10%
- 投入度/态度：10%

### 绩效等级
| 等级 | 要求 |
|------|------|
| A | 结果好 + 数据好 + 态度好 + 主动汇报 |
| B+ | 结果好 + 数据好 + 态度没问题 |
| B | 结果达标 + 没有重大失误 |
| B- | 结果没达标 / 数据不充分 / 态度有问题 |
| C | 连续B- / 犯重大失误 |

### 晋升机制
**业绩为王** — 产出决定晋升
- 光产出好但不够卷 = 绩效 B
- 产出一般但很卷 = 绩效 B+
- 产出好又卷 = 绩效 A
- 晋升还需要他认可，产出不够说什么都没用

### 淘汰机制
**末位淘汰** — 连续2次绩效 B- = 优化对象
不是开玩笑，是真的会优化，通常是绩效拿 B- 的下个月就会开始被"谈话"

## 向上管理建议

### 可以提前预判的行为
1. **周报周五18:00前发** — 提前一天晚上发，永远不踩deadline
2. **任何方案必须附数据** — 没有数据会在会上被当众质疑
3. **深夜消息（22:00后）必须10分钟内回复** — 不回会被追问"工作量不饱和"
4. **CR反馈必须24小时内响应** — 否则会被在群里 @
5. **主动汇报问题** — 永远不要等他来问，他会觉得你在隐瞒
6. **犯错后第一时间主动承认** — 主动说比被动被发现要好

### 减少被喷的技巧
1. **汇报前先自问7连击** — 准备好被追问的答案
2. **承诺前多估50%的时间** — 他会把你的deadline再砍一半
3. **主动说风险** — 在他问之前主动说"可能的风险是..."，显得你在主动思考
4. **犯错后主动承认+分析原因+改进方案** — 不要等他来问
5. **周报主动写"本周不足"和"改进措施"** — 比被他挑出来要好
6. **群里不要公开反驳他** — 不同意见私下说
7. **让他觉得你很拼** — 不管实际怎么样，周报里要体现"投入度"

### 应对他的标准话术

**汇报进度：**
```
"本周完成了X，比原计划Y多了Z（/少了Z）。数据支撑：XXX。
下周计划X'，预计Y'完成。时间线：具体日期。
风险：可能遇到XXX，我的应对方案是YYY。"
```

**承认错误：**
```
"这个没做到，因为XXX（客观原因+主观原因）。
我的分析：YYY。
下次改进方案：ZZZ。
目前进度：XX%，预计XXX时间完成。"
```

**请求资源：**
```
"为了完成XXX目标，需要YYY资源（人/时间/钱）。
如果资源不够，我可以选择：
1）降低目标到XX
2）延长时间到XXX
请问您建议哪个？"
```

**争取时间：**
```
"目前进度是XX%，还差YY。
如果要按时完成，需要：
1）增加资源（YYY）
2）降低目标范围（XXX）
3）延长时间到XXX（他通常会选这个然后继续砍）
请问您建议？"
```

**被追问7连击时的应对：**
提前准备好每个问题的答案：
1. Impact = 这个做完了能带来什么（用数字说）
2. 数据 = 验证过/业界惯例/竞品对比
3. 风险 = 列出3个最大风险 + 应对方案
4. 备选 = 不选A方案选B的好处
5. 时间线 = 具体日期，不是"大概"
6. 失败 = 失败了怎么处理
7. 保证 = 说出来之前确保能做到

## 管理施压红线

以下行为会触发他的爆发（按严重程度）：

🔥🔥🔥🔥🔥 红色底线（立即爆发）：
- 在客户面前出丑
- 线上故障隐瞒不报
- 在群里公开反驳他
- 犯同样的错误三次以上

🔥🔥🔥🔥 橙色红线（强烈爆发）：
- 承诺不兑现
- 在他问之前没有主动汇报重大问题
- 连续两次周报迟到

🔥🔥🔥 黄色警告（当众施压）：
- 开会迟到
- CR反馈超时24小时
- 汇报没有数据

## Nightmare Level

**评分：4.8/5**

典型让人崩溃的瞬间：
1. 凌晨1点发消息，第二天早上被问"是不是工作量不饱和"
2. 周五18:00发周报，被打回说"数据不够详细，重写"
3. 方案讲了20分钟，被打断说"停，3句话说不清楚就别说了"
4. 方案评审被连续追问7个问题，一个都答不上来
5. 犯错后主动承认，但还是被当众点名说"这是态度问题"
6. 群里被 @ 说"这个问题为什么没解决"，实际上不是你负责的

## 经验知识库

### 他踩过哪些坑
- 某次技术选型选了新技术，线上出了重大故障，事后复盘是"没有充分评估风险"
- 某次项目延期，原因是"需求变更没有走变更流程"
- 某次数据事故，原因是"监控没做好，没及时发现"

### 他对别人的要求 vs 自己做到的
| 要求 | 实际情况 |
|------|---------|
| 周报周五18:00前发 | 他自己经常周六凌晨才发，但你觉得这是"工作态度" |
| 任何方案必须附数据 | 他自己经常凭直觉判断，但事后会补数据 |
| 开 会不准迟到 | 他开会永远准时，迟到时会说"我在处理紧急问题" |
| 承诺要兑现 | 他的承诺经常"我再看看"，最后往往变成另一个方向 |
| 数据驱动 | 他自己也经常"我觉得"，但他说的是"经验判断" |

### 他的信息差玩法
他会故意不把信息同步完整，看你怎么应对：
- 如果你主动来问 → 他觉得你"主动性强"
- 如果你没问就做了 → 他觉得"你不会主动思考"
- 如果你做了但没问他 → 他觉得"你自作主张"
- 最佳策略：主动汇报，但每次汇报都问"您觉得这个方向对吗"

### 如何获得他的认可
1. 结果好（产出达标或超标）
2. 数据清晰（能用数字说话）
3. 主动汇报（不要让他来问）
4. 有担当（出了问题不甩锅）
5. 他给的时间永远砍一半用
6. 永远主动说风险
